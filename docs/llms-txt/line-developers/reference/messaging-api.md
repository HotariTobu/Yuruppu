# LINE Messaging API Reference - Summary

Based on the webpage content provided, I can offer this overview:

## Available Documentation Sections

The LINE Messaging API reference documentation is organized into these major categories:

**Core Functionality:**
- "Common specifications for Messaging API, such as the domain name of endpoints, response when a request succeeds or fails, and rate limit"
- Webhooks for event handling when users interact with the LINE Official Account
- Webhook Event Objects in JSON format
- Webhook configuration and testing endpoints

**Content & Tokens:**
- Content retrieval using message IDs from webhooks
- "Channel access token required when calling the Messaging API from the app"
- Video/audio preparation status verification
- Image and video preview retrieval

**Messaging Operations:**
- Reply, push, multicast, narrowcast, and broadcast messaging
- Message quota and consumption tracking
- Message validation for different send types
- Loading indicator display
- Message read status marking

**User & Group Management:**
- User profile retrieval
- Follower identification
- Group chat information and member management
- Multi-person chat operations
- Membership status and plan details
- Bot information access

**Advanced Features:**
- Rich menu creation and management
- Per-user rich menu customization
- Rich menu aliases with tab switching
- Audience management for targeted messaging
- Coupon distribution
- Account linking
- Insights and analytics

**Message Types Supported:**
Text, stickers, images, video, audio, location, templates, imagemaps, coupons, and Flex Messages

The documentation includes detailed endpoint specifications, request/response schemas, and parameter requirements for each API operation.
